1. Single threading va Multi threading


Single Threading Single threading — bu dasturda faqat bitta oqim (thread) ishlaydigan model. Bu shuni anglatadiki, barcha vazifalar ketma-ket (sekvential) bajariladi. Agar bir vazifa uzoq vaqt talab qilsa, dastur boshqa vazifalarni bajarishni kutadi. Misol: JavaScript single-threaded bo'lib, asosiy oqimi (main thread) orqali vazifalarni bajaradi. Multi Threading Multi threading — bu dastur bir vaqtning o'zida bir nechta oqimlar (threads) orqali bir nechta vazifalarni bajaradigan model. Bu, ayniqsa, katta hajmdagi hisoblashlarni yoki kirish/chiqish (I/O) jarayonlarini samarali bajarish uchun foydalidir. Misol: Java yoki Python'dagi threading kutubxonalari multi-threading imkoniyatlarini beradi. Single threading ketma-ket bajarishni talab qiladi, multi threading esa bir vaqtning o'zida bir nechta vazifalarni bajarishga imkon beradi. Multi threading ko'p yadroli protsessorlardan samarali foydalanadi, single threading esa odatda oddiyroq dasturlar uchun mos.



2. Synxron va Asynxron dasturlash

Synxron dasturlash Synxron dasturlashda vazifalar ketma-ket bajariladi. Bir vazifa tugamasdan, keyingisi boshlanmaydi. Masalan, agar bir vazifa uzoq davom etsa, boshqa vazifalar kutish rejimida qoladi. Asynxron dasturlash Asynxron dasturlashda bir vazifa kutish holatida bo'lsa ham, boshqa vazifalar bajarilaveradi. Bu ko'p vaqt talab qiladigan operatsiyalar (masalan, tarmoq so'rovlari yoki fayl o'qish) uchun samarali. Synxron dasturlashda kod ketma-ket bajariladi, asynxron dasturlash esa kutish vaqtlarini boshqarish orqali samaradorlikni oshiradi. Synxron dastur sekinroq ishlashi mumkin, asynxron esa resurslardan samarali foydalanadi.



3. JavaScript Event Loop nima?

Event Loop — JavaScript’ning asynxron xatti-harakatlarini boshqaruvchi mexanizm. U vazifalarni ketma-ket va to'g'ri tartibda bajarilishini ta'minlaydi. JavaScript single-threaded bo'lsa-da, Event Loop yordamida bir vaqtning o'zida ko'plab vazifalarni boshqara oladi. Ishlash prinsipi: Call Stack — Asosiy bajariladigan kodlar joylashtiriladi (masalan, funksiya chaqirilishi). Web API — Asynxron operatsiyalarni (timer, HTTP so'rovlari) bajaradi. Callback Queue — Vazifalar tugagandan so'ng, ularning callback funksiyalari bu yerga tushadi. Event Loop — Call Stack bo'sh bo'lsa, Callback Queue’dagi funksiyalarni Call Stack'ga o'tkazadi.
